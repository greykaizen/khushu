## Vision

Transform the Learn module from 275 lines of hardcoded Kotlin mock data into a
block-driven, offline-first content platform. Each learning card becomes a
long-form article authored in structured JSON, backed by a pre-packaged SQLite
database of authenticated Quran text and Hadith.

---

## Key Decisions (locked, do not relitigate)

| Decision | Rationale |
|---|---|
| Pre-packaged SQLite via `createFromAsset` | Offline-first, instant reads, no runtime API |
| Block-driven JSON content files (not Markdown) | Arabic RTL + TajweedText composable cannot be wedged into a Markdown parser |
| Curated content only — no full Quran/Hadith dump | App is a guided learning path, not a browser. Bundle size stays small |
| LLM drafts content, human verifies | LLMs hallucinate hadith references. `verified: false` on every reference until checked on sunnah.com |
| Fawaz Ahmed `grade === "sahih"` filter enforced in seeder | Programmatic guard against weak narrations entering the app |
| Editorial content stays in JSON (not Room) | Custom duas, explanations, foundations — no external data source exists for these |
| Word-by-word table deferred | 77k words, UI not fully designed. Add in a later phase |
| Hadith browser deferred | No UI spec. Current scope is curated references inside topic articles only |

## JSON Content Block Schema

Each topic is one `.json` file in `assets/learn/`. Template:

```json
{
  "id": "topic_id",
  "title": "Human Readable Title",
  "version": 1,
  "blocks": [
    { "type": "heading",   "text": "Section Title" },
    { "type": "paragraph", "text": "Editorial content..." },
    { "type": "arabic",    "text": "Arabic text", "tajweed": null, "translation": "English" },
    { "type": "ayah",      "surah": 5, "ayah": 6, "display": "Al-Ma'idah 5:6", "verified": false },
    { "type": "hadith",    "collection": "bukhari", "number": 1, "display": "Sahih al-Bukhari, 1", "verified": false },
    { "type": "callout",   "style": "note", "text": "Additional context or madhab note" },
    { "type": "divider" }
  ]
}
```

**Block types:**
- `heading` — section title, renders large
- `paragraph` — editorial prose, LTR English
- `arabic` — standalone Arabic phrase/dua, renders via TajweedText if tajweed != null
- `ayah` — reference to Room ayahs table, renders Arabic + translation inline
- `hadith` — reference to Room hadiths table, renders Arabic (if exists) + English
- `callout` — highlighted note box, styles: "note", "warning", "tip"
- `divider` — horizontal rule between sections

**Rules:**
- Every `ayah` and `hadith` block must have `"verified": false` until manually confirmed
- Every `hadith` and `ayah` block must have a `"display"` field for human-readable label
- `tajweed` field on `arabic` blocks is null until hand-curated markup is added

## Python Toolchain Spec

### `scripts/verify_manifest.py`

**Input:**
- `scripts/data/en.bukhari.json` etc. (Fawaz Ahmed local files)
- All `assets/learn/*.json` topic files

**Behaviour:**
1. Read every topic JSON file
2. For each `hadith` block: look up `collection` + `number` in Fawaz Ahmed JSON
3. Check `grade` field — print ✅ if sahih, ⚠️ if not, ❌ if not found
4. Print first 150 chars of hadith text so human can confirm relevance
5. If grade === "sahih" → flip `"verified": true` and overwrite file
6. For each `ayah` block: confirm surah/ayah exists in Tanzil JSON → flip `"verified": true`

**Output:** Console report + updated JSON files with verified flags

### `scripts/seeder.py`

**Input:**
- `scripts/data/quran-uthmani.json` (Tanzil)
- `scripts/data/en.bukhari.json`, `en.muslim.json`, etc. (Fawaz Ahmed)
- All `assets/learn/*.json` topic files (to know which refs are needed)

**Behaviour:**
1. Parse all topic JSON files, collect all unique (surah, ayah) and (collection, number) references
2. Reject any hadith block where `verified !== true` — hard fail, print error
3. From Tanzil: extract only the referenced ayahs + their surah metadata
4. From Fawaz Ahmed: extract only referenced hadiths, enforce `grade === "sahih"`
5. Build SQLite schema (tables above), insert rows
6. Output `app/src/main/assets/khushu.db`

**Run order:** verify_manifest.py first, seeder.py second (seeder fails on unverified refs)
